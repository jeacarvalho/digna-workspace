### Estratégia de Release - Digna

--------------------------------------------------------------------------------

#### Versões e Mapa de Valor
A estratégia do Digna respeita o tempo social dos Empreendimentos de Economia Solidária (EES). Cada fase entrega não apenas infraestrutura tecnológica, mas também ferramentas pedagógicas de emancipação.

| Versão | Fase | Descrição Sociotécnica |
| ------ | ------ | ------ |
| v0 | Demonstração | Operação Básica, PDV Pedagógico e Validação em Campo |
| v1 | Integração | Transição Gradual Institucional (DREAM -> FORMALIZED) |
| v2 | Finanças | Autonomia Territorial (Moedas Sociais e Bancos Comunitários) |
| v3 | Intercooperação| Escala Nacional e Rede de Apoio Mútuo |

--------------------------------------------------------------------------------

#### v0 - Demonstração Operacional e Validação Cultural
**Objetivo:** Provar conceito técnico, arquitetura offline-first e, criticamente, a *adequação sociotécnica e cultural* da interface junto ao público-alvo.
**Entregas:**
*  [x] Lifecycle Manager (SQLite por tenant para garantia de soberania)
*  [x] Core Lume (Partidas dobradas invisíveis)
*  [x] PDV Interface (Foco pedagógico e linguagem popular livre de jargões)
*  [x] Registro de trabalho (ITG 2002 - Valoração do tempo em int64)
*  [x] Dashboard de Dignidade (Transparência visual para a Assembleia)
*  [ ] **NOVO:** Ciclo de Testes de Usabilidade em Campo com EES reais e ITCPs (Incubadoras Tecnológicas de Cooperativas Populares)

**Critério de Release:**
*  40/40 testes técnicos passando.
*  Operações básicas fluindo perfeitamente offline com sync posterior.
*  **Aprovação de usabilidade e linguagem por pelo menos 3 grupos focais com baixa literacia digital.**

--------------------------------------------------------------------------------

#### v1 - Integração Institucional (Transição Educacional)
**Objetivo:** Oferecer os benefícios da conformidade legal (Compliance) sem burocratizar ou afastar o usuário informal.
**Entregas:**
*  [ ] Módulos educativos embutidos (ex: auxílio na formação de preço no PDV)
*  [ ] Dossiê CADSOL automático (Apenas habilitado quando o grupo atingir maturidade política para formalização)
*  [ ] Geração de documentos oficiais (Atas em Markdown com hash SHA256)
*  [ ] Integração Gov.br (Identidade unificada e simplificada)
*  [ ] Exportação segura para órgãos públicos (com consentimento do grupo)

--------------------------------------------------------------------------------

#### v2 - Finanças Solidárias e Territoriais
**Objetivo:** Suporte a múltiplas unidades de valor, reconhecendo que a riqueza na economia solidária vai além da moeda oficial (Real R$).
**Entregas:**
*  [ ] Integração tecnológica com Bancos Comunitários de Desenvolvimento (BCDs)
*  [ ] Gestão e transação de Moedas Sociais locais
*  [ ] Estoque substantivo (Controle e troca de sementes, animais, horas-trabalho)
*  [ ] Rateio automático de sobras (com painéis de aprovação visual para assembleias)

--------------------------------------------------------------------------------

#### v3 - Intercooperação Nacional
**Objetivo:** Conectar EES isolados em uma rede nacional de apoio e viabilidade econômica.
**Entregas:**
*  [ ] Marketplace B2B restrito a EES (Foco na intercooperação baseada no 6º Princípio do Cooperativismo)
*  [ ] Score de Crédito Social (Baseado em histórico de trabalho solidário, não apenas em capacidade financeira)
*  [ ] Integração de crédito direcionado com BNDES e Serpro
*  [ ] API pública (Open Banking com viés social)
